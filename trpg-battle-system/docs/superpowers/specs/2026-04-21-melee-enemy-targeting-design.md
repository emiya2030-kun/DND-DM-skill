# 近战敌人战斗倾向编排层设计

## 目标

为 `controller=gm` 且属于“近战攻击型敌人”的单位增加一层后端战术编排摘要，让 LLM 不再从全场单位中盲选目标，而是基于后端预筛出的高优先级候选做最终决策。

第一版只解决：

- 近战攻击型敌人的目标筛选
- 候选目标打分与排序
- 返回前二候选目标给 LLM
- 返回每个候选的评分依据与简短原因

第一版不解决：

- 施法型敌人
- 纯特殊动作型敌人
- 多重攻击自动拆解与自动执行
- 移动规划
- 自动替 LLM 拍板最终目标

## 适用边界

该层仅应用于“近战攻击型敌人”。

第一版的近战攻击型敌人定义：

- 当前行动者为敌方单位（`controller=gm`）
- 至少拥有一个近战武器攻击手段
- 当前战术决策主要围绕普通攻击展开

以下对象暂不进入该层：

- 主要靠施法行动的敌人
- 主要靠豁免型特殊动作的敌人
- 召唤/辅助/区域控制型敌人

## 决策流程

### 1. 收集候选目标

候选范围仅考虑本回合可能被近战攻击命中的敌对单位。

第一版采用硬过滤：

- 忽略攻击范围外目标
- 不把范围外目标放入候选列表

这里的“攻击范围”优先复用当前行动者的近战武器可攻击目标集合。

### 2. 硬过滤与强降权

#### 直接排除

- 攻击范围外目标

#### 强降权但不完全排除

- 目标处于黑暗/重度遮蔽中，且攻击者无法看破该黑暗或视野阻碍
- 目标为玩家召唤物

### 3. 权重评分

第一版使用线性加减分，不做复杂公式。

基础分为 `0`。

#### 正向权重

- 当前候选中 AC 最低档：`+30`
- 当前候选中 AC 中低档：`+15`
- 正在维持专注：`+30`
- 对该目标攻击具有优势：`+8`
- 生命值百分比较低：`+10`
- 最大生命值偏低：`+10`

#### 负向权重

- 目标为召唤物：`-25`
- 目标处于黑暗中且无法看破：`-35`
- 若为接敌必须踩入有害地形：`-8`

### 4. 平分规则

若两个候选目标总分接近（差值 `<= 8`），则优先选择正在维持专注的目标。

这条规则用于满足以下预期：

- 一堆脆皮角色里，即使有一个高防御维持专注的角色，也大概率攻击它

## 用户偏好到规则的映射

### 1. 优先攻击低防御或对其攻击有优势的角色

- 低 AC 给予高权重
- 攻击优势仅作小幅加分，不压过 AC 和专注

### 2. 敌人可以承受不利地面效果继续战斗

- 有害地形不是硬阻止项
- 仅给予小幅负分

### 3. 黑暗中的脆皮不是默认首选

- 若目标在黑暗/重度遮蔽中，且攻击者看不破，则强降权
- 若攻击者具备看破黑暗的能力，则取消该惩罚

### 4. 高防但专注的角色也会被优先考虑

- `is_concentrating` 高权重
- 总分接近时由专注目标胜出

### 5. 忽略攻击范围外的对手

- 范围外直接不进入候选

### 6. 召唤物权重低

- `category == summon` 负权重

## 判定字段

第一版尽量直接复用现有 encounter state 或实体运行时数据：

- `in_attack_range`
  - 由当前行动者近战攻击可命中集合提供
- `distance_feet`
  - 当前行动者到候选目标距离
- `ac`
  - 候选目标当前 AC
- `hp_ratio`
  - `current_hp / max_hp`
- `max_hp`
  - 用于判断“血薄”
- `is_concentrating`
  - 根据实体当前专注状态或持续法术状态得出
- `is_summon`
  - `category == summon`
- `attack_has_advantage`
  - 复用攻击请求层已有的优势判断逻辑，而不是让 LLM 自行猜测
- `blocked_by_darkness`
  - 目标是否在攻击者看不破的黑暗/重度遮蔽中
- `terrain_cost_note`
  - 说明接敌是否需要穿过有害地形

## 输出结构

第一版计划在 `current_turn_context` 下新增：

- `enemy_tactical_brief`

结构如下：

```json
{
  "role": "melee_attacker",
  "can_act": true,
  "candidate_targets": [
    {
      "entity_id": "ent_ally_miren_001",
      "name": "米伦",
      "score": 46,
      "in_attack_range": true,
      "distance_feet": 5,
      "ac": 16,
      "hp_ratio": 0.81,
      "is_concentrating": false,
      "is_summon": false,
      "attack_has_advantage": false,
      "blocked_by_darkness": false,
      "terrain_cost_note": null,
      "score_breakdown": [
        {"rule": "low_ac_top_tier", "value": 30},
        {"rule": "low_max_hp", "value": 10},
        {"rule": "in_range", "value": 6}
      ],
      "reason_summary": "AC 较低，生命上限较低，且已在近战范围内。"
    }
  ],
  "llm_decision_note": "优先从前二候选中选择最终攻击目标；若无额外战术理由，不建议攻击召唤物或黑暗中不可见目标。"
}
```

第一版要求：

- 仅返回排序后的前二候选目标
- 每个候选都带 `score_breakdown`
- 每个候选都带 `reason_summary`
- 不自动生成动作执行结果
- 不直接替 LLM 选择最终目标

## 与现有结构的关系

第一版采用增量方式：

- 保留现有 `current_turn_context.actor`
- 保留现有 `battlemap.entities`
- 新增 `current_turn_context.enemy_tactical_brief`

这样前端与既有旧结构不受影响，LLM 可逐步改读新字段。

## 第一版暂不做的内容

- 多重攻击拆解为具体攻击序列
- 特殊动作（如吸取生命）的执行接口
- 远程敌人的目标选择模型
- 施法型敌人的目标选择模型
- 移动路径与位移收益比较
- 更高级的战术模板与行为树

## 验收标准

第一版完成后，应满足：

- 近战攻击型敌人回合内，后端能返回前二候选目标
- 候选排序符合用户定义的 6 条近战倾向
- 召唤物默认不进入前二，除非可选目标极少
- 黑暗中不可见目标默认不会进入前二，除非攻击者看破黑暗
- 高 AC 但维持专注的角色，在分差接近时能压过普通脆皮
- 输出结构对 LLM 可直接阅读，不需要自行重新计算全场威胁
