# Move Encounter Entity 设计说明

## 目标

新增一个专门的 `move_encounter_entity` 能力,用来执行带规则校验的遭遇战移动.

这个工具必须做到:

- 把某个实体移动到遭遇战地图中的目标格子
- 按"逐步路径"校验,而不是只检查终点
- 计算移动力、疾跑、困难地形、墙体和占位碰撞
- 通过体型规则支持多格生物
- 继续保持 battlemap 只是 encounter 状态的只读投影

这份设计明确不包含地图点击交互.地图仍然只是战斗状态的展示层,不是状态真相源.

## 推荐方案

把 `move_encounter_entity` 实现成一个新的独立服务,而不是继续扩展现有偏 CRUD 的 `EncounterService`.

原因:

- 移动规则本身已经和实体的增删改查不是同一层职责
- 独立后更容易测试,也更方便后续扩展
- 以后强制位移、传送、击退、拖拽都可以复用同一套移动规则层

## 范围

本次包含:

- 新的移动服务
- `EncounterEntity` 新增 `size`
- 基于锚点坐标展开占格区域
- 为后续触及/距离计算提供中心点派生能力
- 逐步路径校验
- 直线与斜线移动消耗
- 疾跑支持
- 非计费移动支持
- 地形与占位规则校验
- 完整测试覆盖

本次不包含:

- 地图点击输入
- 动画
- 触及和攻击规则改造
- 传送类特例
- 挤压通过(squeezing)
- 借机攻击

## 数据模型

### EncounterEntity

为 `EncounterEntity` 增加 `size` 字段.

允许值:

- `tiny`
- `small`
- `medium`
- `large`
- `huge`
- `gargantuan`

默认值:

- 老数据如果没有这个字段,一律按 `medium` 处理

### position 的语义

`position` 继续作为存储层的事实源,表示实体的**左下角锚点格**.

示例:

- `medium` 在 `(10, 10)`,只占 `(10, 10)`
- `large` 在 `(10, 10)`,占 `(10,10) (11,10) (10,11) (11,11)`

### 运行时派生辅助信息

以下内容由规则层计算,不作为独立事实源持久化:

- `occupied_cells`
- `center_position`

## 体型与占格规则

体型对应占格:

- `tiny` / `small` / `medium` -> `1x1`
- `large` -> `2x2`
- `huge` -> `3x3`
- `gargantuan` -> `4x4`

合法性要求:

- 路径上的每一步,都必须形成一个完全合法的占格区域
- 终点占格区域中的每一格都必须在地图范围内
- 终点占格区域中的每一格都不能碰到墙
- 必须采用严格校验,不允许只检查最终结果

## 移动规则

### 工具输入

建议的服务签名:

```python
move_encounter_entity(
    encounter_id: str,
    entity_id: str,
    target_position: dict[str, int],
    count_movement: bool = True,
    use_dash: bool = False,
)
```

### 基础消耗

- 上下左右移动一格消耗 `5` 尺
- 斜线移动按"连续斜线"规则交替计费:
  - 连续段中的第一次斜线消耗 `5`
  - 连续段中的第二次斜线消耗 `10`
  - 一旦中间插入直线移动,斜线计数重置

例子:

- 斜 -> `5`
- 再斜 -> `10`
- 直 -> `5`
- 再斜 -> `5`

### 疾跑

- `use_dash=True` 时,本次可用移动额度增加 `speed.walk`
- 这个变化只用于当前这次移动校验
- 本阶段不处理动作经济是否已经使用的问题

### 非计费移动

当 `count_movement=False` 时:

- 仍然必须校验路径是否合法
- 仍然计算移动成本,方便回报结果
- 但不扣减 `speed.remaining`

## 地形规则

### 墙

- 不能进入
- 不能穿过
- 对于多格生物,只要任意一个占格触碰墙体,该步就非法

### 困难地形

- 进入困难地形格时,该步移动消耗翻倍
- 对于多格生物,只要这一移动步骤中新进入的占格里有任意困难地形,该步按双倍处理

## 占位规则

### 同伴

- 可以穿过同伴占据的格子
- 默认不能停在已被同伴占据的格子里
- 例外:如果目标格里的同伴是 `tiny` 或 `small`,允许共享终点格

### 敌人

- 不能穿过敌人占据的格子
- 也不能把敌人占据的格子作为终点

### 多格生物

上述同伴/敌人规则,都是针对完整占格区域判断,不是只看锚点格.

## 路径校验策略

### 路径生成

从当前锚点到目标锚点生成离散路径,一次只移动一步.

路径生成器必须支持:

- 直线步
- 斜线步
- 保留步骤顺序,方便后续正确计算 5/10 斜线消耗

### 每一步的校验顺序

对每一步执行:

1. 先把锚点移动一步
2. 根据体型展开新的占格区域
3. 校验地图边界
4. 校验墙体
5. 校验占位
6. 结合斜线规则和地形规则计算这一步移动消耗
7. 通过则继续,失败则立即终止

## 服务输出

服务主返回值应该仍然是更新后的 `Encounter`.

同时建议内部或伴随返回一份结构化移动结果,便于调试和未来工具化展示.

建议字段:

- `path`
- `feet_cost`
- `used_dash`
- `movement_counted`
- `blocked_reason`

## 失败行为

当移动非法时:

- 抛出明确的 `ValueError`
- encounter 状态不得被修改

典型错误原因:

- 目标超出地图边界
- 被墙阻挡
- 被敌方占位阻挡
- 终点被占据,无法停留
- 剩余移动力不足

## 架构拆分

建议新增文件:

- `tools/services/encounter/move_encounter_entity.py`
- `tools/services/encounter/movement_rules.py`

职责划分建议:

- `move_encounter_entity.py`
  - 负责 orchestration
  - 负责读取/保存 encounter
  - 负责最终应用状态更新
- `movement_rules.py`
  - 负责体型占格展开
  - 负责中心点推导
  - 负责路径生成
  - 负责移动消耗计算
  - 负责碰撞和地形校验

这样规则逻辑和持久化 orchestration 可以保持解耦.

## 测试要求

至少覆盖以下场景:

- 合法直线移动
- 合法斜线移动,连续斜线按 5/10 交替计算
- 中间插入直线后,斜线计数会重置
- 越界移动失败
- 撞墙失败
- 穿过敌人失败
- 穿过同伴成功
- 默认不能停在同伴格
- `tiny` / `small` 同伴允许共享终点格
- 困难地形增加移动消耗
- 剩余移动力不足失败
- `use_dash=True` 后原本不合法的移动变合法
- `count_movement=False` 时不扣剩余移动力
- 大型生物占格碰撞失败
- 大型生物占格碰墙失败
- 大型生物中途步骤严格校验

## 集成说明

- 第一阶段不要求修改 `GetEncounterState` 的整体结构
- battlemap 继续只读取更新后的 entity 位置进行渲染
- 后续如果要做路径预览或移动动画,可以在这一层之上追加,但不是这次服务的责任

## 决策总结

- `EncounterEntity` 增加 `size`
- 存储层继续使用左下角锚点
- 运行时派生 `occupied_cells` 和 `center_position`
- 使用严格占格校验
- 同伴可穿过
- 敌人不可穿过
- 只有 `tiny` / `small` 同伴允许共享终点格
- 斜线消耗使用"连续斜线 5/10,直线重置"的规则
- 移动规则单独放进新的服务切片中
